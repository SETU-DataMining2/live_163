Kaggle Competition

Overview

Customer churn is the name used when a customer or subscriber terminates their ongoing service with a company. The churn rate refers to the proportion of contractual customers or subscribers who leave a supplier during a given time period. It is an important metric because it is a possible indicator of customer dissatisfaction, or indicator of external factors such as cheaper and/or better offers from the competition, etc.

The aim of this assignment is to build a model capable of accurately predicting that a customer will churn based on the information available in the dataset.

In addition, this assignment will be managed by Kaggle. Kaggle is an online community platform for data scientists and machine learning enthusiasts. It allows users to collaborate with other users, find and publish datasets, use GPU integrated notebooks, and compete with other data scientists to solve data science challenges. Kaggle supports semi-private competitions where you can test your model performance against others taking this module.

You'll need a private key to access the competition. This key can be found under the welcome message in the module page in Moodle.

Details

  • First create a Kaggle account (using whatever account name/details you wish) and read the competition instructions (Please ask if there is anything that is unclear).
  • Setup a suitable ML project folder (see above).
  • Download the train and test datasets — three each.
  • Download the sample submission file.
  • Split *_train files into your train and test datasets.
  • Train a classifier using the workflow covered during the course.
  • Using your model, generate predictions for the data in *_test files and upload to get on top of the leaderboard.

Marking Scheme

Within the assignment, marks are assigned as follows. (Please note that breakdown may be subject to slight changes (±5%).):

  1. Notebooks consistent with specification (5%)
  2. Exploratory data analysis (10%)
  3. Model performance (65%)
  4. Details of the implemented model and steps attempted to improve to model (20%)

Advice

Build a model as soon as possible

I'm a big fan of getting to a baseline model that produces predictions in a valid format as quickly as possible. This is important, in general, because a baseline model is a good indicator of possible issues, what steps need to be improved, and how must effort one should invest. In addition, since this assignment is run via Kaggle you want, as soon as possible, ensure that you understand the submission and evaluation process. Both of us are here to help you on this.

Exploratory Data Analysis

  • Don't just generate plots/statistics without commenting on their importance/consequence.

How to maximise your marks

  • Clear, informative EDA.
  • Model performance using specified metric (recall).
  • Clear/rational model building process.
