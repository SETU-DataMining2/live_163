Install / Verify a Suitable Python Environment

Contents

  1. Installation
    We cover installation of python using either a full anaconda distribution (about 5.7 GB) or a reduced miniconda distribution (about 248MB).
  2. Extra Packages
    In addition to the main python data analysis modules of pandas, scikit-learn, etc., we will use numerous other modules which will be listed here.

Installation

While any Python 3.9 distribution should be suitable for our needs, I recommend you install the full anaconda distribution (about 5.7 GB) or a reduced miniconda distribution (about 248MB).

Which should you choose? To be honest, it doesn't really matter but with miniconda you will have faster initial download and less hard disk space used, but you will find that you will need to install extra packages more frequently than an anaconda user. And under windows — with its crippled shell — the anaconda installation will be easier to use.

It is possible to have multiple python distributions installed on your system — you just have to manage the system paths so to allow you to switch between.

Finally, there are other options — all of which should be suitable for this course's needs. However, while I'm happy to help sort out issues the priority will be sorting issues in anaconda or miniconda.

  • Standard Python from Python Software Foundation.
  • On OS X using homebrew of MacPorts, see Thoughts on macOS Package Managers.
  • Enthought Deployment Manager (EDM).

Anaconda (Full python distribution)

To install anaconda:

  • Pick any suitable 64-bit, Python 3.9 installer for you system from anaconda.
  • Run installer and default options should work for your system. Under OS X, I typically install into my home directory, so will end up with a folder ~/opt/anacond3.

Miniconda (python distribution)

To install miniconda:

  • Pick any suitable 64-bit, Python 3.9 installer for you system from miniconda.
  • Run installer and default options should work for your system. Under OS X, I typically install into my home directory, so will end up with a folder ~/opt/miniconda3.

Verify Installation

Regardless of anaconda/miniconda option you picked, once installed you should be able to run python from the command line 

1
python

to get output and python prompt (based on a new/clean installation of anaconda python on Jan 16 2023).

1
2
3
4

[Clang 12.0.0 ] :: Anaconda, Inc. on darwin

>>>

Type 1+1 at the prompt and press enter. You should get 2 or we are in trouble.

1

>>> 1+1
2
>>> quit()

Use quit() to exit the python shell and return to the command prompt.

Now, while at the command prompt, you can installed python packages as covered next.

Extra Packages

Both anaconda and miniconda use a package manager called conda. To install a package you typically use command 

1
conda install -y PACKAGE_NAME

The -y option causes the install to automatically carry out required installs/updated instead of asking for user confirmation.

Note: some packages are not installable using conda, in these cases or if your are using a standard (non-anaconda variant) distribution you can install using the pip installer using 

1
pip install PACKAGE_NAME

Side Note: Environments

Both conda and pip support the idea of environments where within the one python distributions you might have a multiple of environments. This is to help avoid version conflicts between modules. aanconda and miniconda both start with one default environment called root. If you wish to use environments, you need to apply two steps: 1) create an environment using 

1
conda create -n data_mining

and 2) activate it using 

1
activate data_mining

Since all of the packages that we plan to use work are (currently, Jan 2023) compatible we won't need to use multiple environments but they are a good idea and if you are interested there is a nice introduction at Happy Belly Bioinformatic.
